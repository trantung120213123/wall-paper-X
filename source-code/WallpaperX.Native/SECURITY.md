# Security Policy

## Supported Versions

Security fixes are applied to the latest `main` branch.

## Reporting a Vulnerability

Please do not open a public issue for security vulnerabilities.

Instead, report privately to the project maintainer with:

- A clear description of the issue
- Reproduction steps
- Potential impact
- Suggested mitigation (if available)

## Response Expectations

| Item | Target |
|---|---|
| Initial response | Within 7 days |
| Fix timeline | Depends on severity/complexity |

## Scope

Security-related examples:

- Privilege escalation in installer flow
- Unsafe path handling in file operations
- Command/script injection opportunities
